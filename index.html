📋 Setup Guide — Your Hardware & Software Stack

🔍 Step 1: Discover What Your Inverter Can Do Do This First

🔎 Identify Your Inverter Brand

1
Find the model sticker on the side of your inverter. Common WiFi brands in India: Growatt, Deye, Sofar, SolarEdge, Goodwe, Solis, Voltronic
2
Check if there's a WiFi stick/dongle plugged into the inverter (usually labeled "WiFi Logger", "Data Stick", or "WLAN")
3
Look for the inverter's own WiFi hotspot on your phone — many broadcast as "AP_XXXXXX" or brand name when in setup mode
4
Check your home router — find the inverter's IP address in connected devices list. Try opening that IP in browser to see a local web portal

📡 Scan For Open Ports (on Pi4)

Run these commands on your Raspberry Pi 4 to detect what your inverter exposes over WiFi:

# Install nmap (network scanner)
sudo apt install nmap -y

# Find inverter on network
nmap -sn 192.168.1.0/24

# Scan inverter ports (replace IP)
nmap -p 502,8899,8888,80,443,23 192.168.1.XXX

Port 502 = Modbus TCP   Port 8899 = SolarmanV5   Port 80 = Web UI

🛠️ Step 2: Hardware Wiring Guide Your Devices: ESP32 + Pi4 + UNO

📶 Option A — WiFi Only (Easiest, Zero Wiring)

1
Pi4 connects to home WiFi → pings inverter WiFi logger → reads data via SolarmanV5 or Modbus TCP. No wires needed if your inverter has WiFi stick!
2
ESP32 connects to home WiFi independently → posts sensor data (PZEM-004T load readings) to Pi4 via MQTT
3
Pi4 runs all software: MQTT broker (Mosquitto) + Node-RED + InfluxDB + Grafana — accessible from mobile/laptop on same WiFi
# Test WiFi inverter connection
pip3 install pysolarmanv5
python3 -c "from pysolarmanv5 import PySolarmanV5;
  inv = PySolarmanV5('192.168.1.X', 1234567890);
  print(inv.read_holding_registers(0,10))"

🔧 Option B — ESP32 + RS485 (If No WiFi Stick)

ESP32 PinRS485 ModuleColor

RS485 Module → Inverter RS485 port:
A+ pin on module → A on inverter
B− pin on module → B on inverter
⚠ Swap A/B if no data — it's normal to get it reversed!

⚡ PZEM-004T Load Meter (Clamp)

1
Clamp the CT coil around your main live wire at the distribution board — measures real power, voltage, current, power factor
2
Connect PZEM to ESP32 via TX/RX pins. Library: PZEM004Tv30 on Arduino IDE
3
UNO tip: Use Arduino UNO only for quick sensor prototyping — it lacks WiFi. Use ESP32 for all WiFi-connected sensors
PZEM-004T → ESP32:
PZEM TX → GPIO16 (ESP RX)
PZEM RX → GPIO17 (ESP TX)
5V → VIN
GND → GND

💻 Step 3: Software Stack on Raspberry Pi 4 All Free & Open Source

🐳 Install Everything with Docker

1
Install Docker + Docker Compose on Pi4 — this runs all services in containers, easiest approach
# Install Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker pi

# Create docker-compose.yml
mkdir ~/solar && cd ~/solar
2
Services to run: Mosquitto (MQTT broker) + Node-RED (visual data pipeline) + InfluxDB (database) + Grafana (dashboard)
3
Access everything from your phone/laptop on the same WiFi: Pi4 IP:3000 = Grafana dashboard | Pi4 IP:1880 = Node-RED flows

🔄 Data Pipeline Flow

1
Python script on Pi4 polls inverter every 30 seconds via WiFi → publishes to MQTT topic solar/generation
2
ESP32 reads PZEM-004T every 5 seconds → publishes to MQTT topic solar/load
3
Node-RED subscribes to MQTT → calculates efficiency → writes to InfluxDB → sends Telegram/WhatsApp alert when panels need cleaning
4
Grafana queries InfluxDB → live dashboard accessible on mobile browser or as PWA (works like an app!)

🧹 Panel Cleaning Algorithm

1
Call OpenWeatherMap API (free) to get solar irradiance / cloud cover for your location
2
Calculate expected output = 3500W × (irradiance factor) × time_of_day_curve
3
If actual / expected < 0.80 on a clear day → flag as cleaning needed. Node-RED counts 3 consecutive days to avoid false alarms
4
Alert via Telegram Bot (free!) → "🧹 Solar panels need cleaning! Yield is 25% below expected for 3 days."
efficiency = actual_kw / expected_kw
if efficiency < 0.80 and is_clear_day:
  consecutive_low_days += 1
if consecutive_low_days >= 3:
  send_telegram_alert()
